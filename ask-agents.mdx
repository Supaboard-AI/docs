**Ask Agents** is Supaboard's conversational interface. Type a question in plain English and Agents queries your data, then returns charts, tables, KPIs, and written explanations — no SQL required.

Ask Agents is powered by your workspace's [Agents](./Agents.mdx). Each agent defines what data agents can access and how it interprets your questions.

---

## Opening Ask Agents

Click **Ask Agents** in the sidebar. If you have an existing conversation, it opens where you left off. To start fresh, click **New Conversation**.

## Conversations

### Conversation list

The left panel shows all your conversations, grouped by recency:

- Today
- This Week
- Last Month
- Older

**Search** to filter by conversation name. Conversations load 15 at a time — scroll to load more.

### Starting a new conversation

Click **New Conversation**. When you send your first message, you select which agent should answer it. Each conversation stays tied to that agent for its lifetime.

### Conversation limit

Each conversation supports up to **20 exchanges**. When the limit is reached, the input is disabled. Start a new conversation to continue.

## Asking a Question

Type your question in the input bar at the bottom: _"Ask anything about your data..."_ and press Enter.

### Query modes

Use the **mode selector** in the input bar to control how Agents processes your question:

#### Concise

Quick follow-ups and simple lookups where speed matters more than depth. Answers the "what" questions.

#### Deep Dive

Queries that need thoughtful analysis — charts, tables, KPIs, with exploratory insights that go deep into understanding what is going on with your data. Answers the "how" and "why" questions.

### Additional input options

| Option          | What it does                                                                               |
| --------------- | ------------------------------------------------------------------------------------------ |
| **Attach file** | Upload a PDF, CSV, Excel, or image to give Agents extra context for this specific question |
| **Web search**  | Toggle on to let Agents supplement its answer with live web results                        |
| **Tools**       | Select MCP tools (external integrations) for Agents to call when composing its answer      |

## Understanding Responses

Every response appears as a message bubble with multiple tabs you can switch between.

### Answer tab

The primary response. Depending on your question, this may contain:

| Response type | What it looks like                                               |
| ------------- | ---------------------------------------------------------------- |
| **Text**      | A written, markdown-formatted explanation                        |
| **KPI**       | A headline metric card with an optional trend indicator          |
| **Table**     | A paginated, scrollable data table                               |
| **Chart**     | An interactive visualisation — bar, line, pie, scatter, and more |

A single response can contain multiple widgets alongside explanatory text.

### Source tab

Shows exactly which data sources and tables were queried to produce the answer. Use this to verify where the numbers came from.

### Definitions tab

Key terms or metrics extracted and defined by Agents for added clarity.

### Steps tab

The reasoning steps Agents took to arrive at its answer — useful when you want to understand _how_ a result was calculated, not just _what_ it is.

### Message actions

Every message has a small toolbox:

| Action       | What it does                                                           |
| ------------ | ---------------------------------------------------------------------- |
| **Share**    | Copy a link to this specific message                                   |
| **Copy**     | Copy the message content to the clipboard                              |
| **Feedback** | Thumbs up or thumbs down — helps improve the agent's answers over time |

## Pinning Widgets to a Dashboard

Charts, tables, and KPIs generated by Agents can be saved directly to any dashboard — and they stay live, re-querying your data on every dashboard load.

### How to pin

1. After Agents responds, widgets appear in the **right-side panel**.
2. Check the widgets you want to keep.
3. Click **Pin to Dashboard**.
4. In the modal that opens, either:
   - **Select an existing dashboard** from the list, or
   - **Create a new dashboard** inline.
5. Confirm — the widgets are added to the dashboard immediately.

## Managing Conversations

All conversation management is in the **topbar** — the three-dot menu next to the conversation name.

### Rename

Opens a modal with a text input. The default name is _"Untitled Conversation"_ until renamed. Type a new name and save.

### Delete

Opens a confirmation modal. Deleting a conversation permanently removes it and all its messages. **This cannot be undone.**

### Control access

Conversations are private by default — only you can see them. Use **Control Access** to share a conversation's history with teammates.

- All workspace members are listed with checkboxes.
- Your entry and the workspace Owner are always included and cannot be removed.
- Toggle any other user to grant or revoke access.

---

## Permissions

### Who can open Ask Agents

Access is controlled by the **Agents feature flag** on each user's workspace membership. Users without this flag cannot open Ask Agents (see [Workspaces — Feature Access](./Workspaces.mdx#6-feature-access)).

### Who can use a specific agent

Each agent has its own User Access list. If an agent is restricted, only users on that list can select it when starting a conversation. Unrestricted agents are available to everyone with Agents access.

### Conversation access

Conversations are private by default. Share them explicitly via [Control Access](#6-managing-conversations).
